# Active Design Skills

Apply these design skills to ALL UI work in this project, without being asked:

1. **/impeccable** — Load `.impeccable.md` at project root first. Enforce the project design language (brand, tone, typography, spacing, components) on every UI task.
2. **/frontend-design** — Commit to a distinctive aesthetic direction (typography, color, spacing) before writing any code. Never default to generic AI aesthetics.
3. **/ui-ux-pro-max** — Apply sector-appropriate visual style. Match the UI to the domain and audience.
4. **`/ui [description]`** — Use the 21st.dev Magic MCP (`@21st-dev/magic`) for all new component generation.
5. **/accessibility-review** — Run the `accessibility-lead` agent on every generated UI component before considering it complete.

These are non-negotiable defaults. Do not skip any step even if the user does not mention them.
